# Навыки, требования и маркеры: Data & AI Platform Leader / CDO-CTO

## Ключевые навыки для HH

Data & AI, CDO, CTO, Data Platform, DWH, Data Lake, MDM, Data Governance, Data Quality, Metadata, Data Catalog, Lineage, DataOps, MLOps, BI, SQL, AI/ML, modern data stack, Spark/Flink, MPP, Kafka, NiFi, Airflow, cloud/on-prem, legacy modernization, enterprise architecture, управление распределенными командами, финансовый сектор

## Требования, которые должен закрывать верх CV

- Формирование data strategy и технологической стратегии платформы данных/ИИ.
- DWH/Data Lake, Data Governance, MDM, метаданные, качество данных, витрины.
- End-to-end ландшафт данных: источники, интеграция, хранение, обработка, аналитика, потребители.
- Modern data stack: Spark/Flink, MPP, Kafka, NiFi/Airflow, S3/Hadoop, cloud/on-prem.
- DataOps/MLOps, безопасная работа с данными, политики доступа, lineage/catalog.
- Управление командами инженеров, аналитиков, архитекторов, разработчиков и консультантов.
- Прагматичная модернизация legacy и приоритизация в условиях ограниченного бюджета.
- Работа с руководителями бизнеса, P&L/бюджетами и измеримым эффектом от data/AI инициатив.

## Доказательства, которые стоит поднимать выше

- IBM Data & AI: руководство профессиональными услугами, внедрением платформ данных, аналитических решений и AI-oriented инициатив для крупных заказчиков.
- Опыт проекта для крупной государственной финансовой организации: платформа сбора данных из территориально распределенных источников, анализ и предоставление данных приложениям, аналитическим системам и пользователям.
- Астра: трек по формированию направления продуктов для управления данными и искусственного интеллекта; проработка слоев modern data platform.
- Teradata: работа с enterprise data platform / MPP DWH-повесткой для крупных корпоративных клиентов.
- MDM/Data Governance: проекты SAP MDM в банковском секторе и телекоммуникациях.
- Управление распределенными командами до 50 человек: консультанты, инженеры, аналитики, разработчики, архитекторы.
- P&L, бюджетирование и управление портфелем проектов/сделок совокупным объемом 20+ млн долларов.

## Что ослаблять

- Не выглядеть только консультантом или продавцом: поднимать архитектуру, governance, команду и delivery.
- Не приписывать hands-on владение всеми инструментами modern data stack; формулировать как архитектурное и управленческое понимание.
- Для CTO-вакансий усиливать инженерные команды, legacy modernization и целевую архитектуру; для CDO-вакансий — strategy, governance и business value.
