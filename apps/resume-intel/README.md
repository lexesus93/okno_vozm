# Resume Intel MVP

Локальное приложение для разбора писем вида «Ваше резюме привлекло внимание» из hh.ru / LinkedIn и сопоставления интереса компании с типами CV.

## Что умеет MVP

- Импортировать письмо через drag-and-drop `.eml`, `.txt`, `.html`.
- В Electron-режиме принимать прямой drag из Apple Mail: если Mail не отдаёт файл браузерному drop, приложение читает выбранное письмо через native bridge.
- Парсить источник письма, тему, компанию, возможное название резюме и тип события.
- Сохранять события в локальную SQLite-базу.
- Читать 3 CV-типа из `output/cv_types`.
- Вести отдельный справочник текущих опубликованных HH-резюме, чтобы не путать их с проектными CV-типами.
- Считать первичное пересечение письма и вакансий компании с CV-типами.
- Показывать рекомендации, какие пункты соответствующего CV стоит усилить.
- Позволять вручную добавить текст вакансии компании, чтобы matching стал точнее.

## Запуск web-режима

Из папки приложения:

```bash
cd apps/resume-intel
docker compose up --build
```

Открыть UI:

```text
http://localhost:5177
```

Backend API:

```text
http://localhost:8787
```

SQLite-база хранится локально:

```text
apps/resume-intel/data/resume_intel.sqlite3
```

## Текущие HH-резюме

3 CV-типа в `output/cv_types` — это проектные целевые версии, а не обязательно то, что уже опубликовано в HH.

Чтобы приложение понимало, на какое текущее HH-резюме могла прийти реакция, заполните:

```text
apps/resume-intel/config/hh_resumes.json
```

Минимально нужно заменить `title` на точные названия резюме из `https://hh.ru/applicant/resumes`. Ссылка требует вашей авторизации в браузере, поэтому MVP не читает её автоматически.

## Native Electron-режим для прямого drag из Mail

Web-режим принимает обычные файлы, но Apple Mail на macOS иногда отдаёт письмо как native drag object, а не как `.eml` файл. В таком случае браузер видит drag-over, но при отпускании мыши не получает файл.

Для этого добавлен Electron-режим:

```bash
cd apps/resume-intel
docker compose up -d backend
cd frontend
npm install
npm run electron:dev
```

Откроется отдельное окно `Resume Intel`. В него можно перетаскивать письмо прямо из списка Apple Mail.

Как это работает:

1. Если Mail отдаёт обычный файл/HTML/text payload, UI импортирует его напрямую.
2. Если payload пустой, Electron вызывает native bridge.
3. Native bridge через AppleScript/JXA читает текущее выбранное письмо в Apple Mail.
4. Письмо отправляется в backend как событие внимания.

При первом запуске macOS может спросить разрешение на управление Mail для приложения Electron/Terminal. Это нужно, чтобы прочитать выбранное письмо.

Если прямой drag не сработал, проверьте:

- письмо действительно выделено в Mail;
- Mail открыт и активен до начала перетаскивания;
- macOS разрешил Automation-доступ к Mail;
- backend запущен на `http://localhost:8787`.

## Альтернативный импорт через Finder

1. Откройте письмо в Mail.
2. Перетащите письмо из списка писем на рабочий стол или в Finder. Обычно Mail сохранит `.eml`.
3. Перетащите получившийся файл в область импорта Resume Intel.

Если drag-and-drop из Mail сохранит не `.eml`, можно открыть письмо, скопировать текст и сохранить его как `.txt`; MVP также принимает `.txt` и `.html`.

## Рабочий сценарий

1. Импортировать письмо «Ваше резюме привлекло внимание».
2. Проверить, какую компанию и опубликованное HH-резюме определил парсер.
3. Посмотреть лучший проектный CV-тип и рекомендации.
4. Если у компании есть интересная вакансия, вставить ее текст в форму «Вакансия компании».
5. После сохранения вакансии matching пересчитается с учетом требований вакансии.

## Ограничения MVP

- Нет IMAP и автоматического чтения почты.
- Парсер писем эвристический: если HH изменит шаблон, компанию/резюме иногда придется уточнять вручную в будущей версии.
- Автоматического поиска вакансий компании пока нет; вакансии добавляются вручную текстом.
- Скоринг простой keyword-overlap, без LLM/embeddings. Этого достаточно для первого локального triage, но позже можно заменить на семантический анализ.
